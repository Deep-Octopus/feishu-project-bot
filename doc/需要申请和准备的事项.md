# 项目启动前需要申请和准备的事项

## 一、飞书开放平台（必须）

### 1.1 创建企业自建应用

**操作步骤：**
1. 登录 [飞书开放平台](https://open.feishu.cn/)，使用企业管理员账号
2. 进入「开发者后台」→「创建应用」→「企业自建应用」
3. 填写应用名称（如：项目进度助手）、描述、上传图标
4. 创建完成后记录以下信息：

| 配置项 | 说明 | 填写到 .env |
|--------|------|------------|
| App ID | 应用唯一标识，格式 `cli_xxxxxxxx` | `FEISHU_APP_ID` |
| App Secret | 应用密钥 | `FEISHU_APP_SECRET` |

### 1.2 申请机器人权限

在应用详情页 → 「权限管理」，申请以下权限：

**消息权限（必须）：**
- `im:message` — 获取与发送单聊、群组消息
- `im:message:send_as_bot` — 以应用身份发消息
- `im:message.group_at_msg:readonly` — 接收群 @ 消息

**用户信息权限（必须）：**
- `contact:user.base:readonly` — 获取用户基本信息

**群组权限（必须）：**
- `im:chat:readonly` — 获取群组信息
- `im:chat.member:readonly` — 获取群成员信息

**文件权限（可选，用于导出功能）：**
- `drive:drive` — 访问云空间文件

### 1.3 配置事件订阅

在应用详情页 → 「事件订阅」：

1. **加密策略**：选择「AES 加密」，记录：
   - Verification Token → `FEISHU_VERIFICATION_TOKEN`
   - Encrypt Key → `FEISHU_ENCRYPT_KEY`

2. **请求地址**：填写服务器回调地址（需要服务器先部署好）：
   ```
   http://your-server-ip/api/v1/feishu/callback
   ```
   > 注意：飞书要求回调地址必须是公网可访问的 HTTPS 或 HTTP 地址

3. **订阅事件**：添加以下事件：
   - `im.message.receive_v1` — 接收消息

### 1.4 配置机器人

在应用详情页 → 「机器人」：
- 开启机器人功能
- 设置机器人名称和描述

### 1.5 发布应用

- 提交审核（企业内部应用通常由管理员直接审核）
- 审核通过后，将机器人添加到目标群聊

### 1.6 外部人员权限（如需支持外包/供应商）

- 联系飞书企业管理员，在「管理后台」→「安全」→「外部协作」中开启外部人员权限
- 在应用权限中申请 `contact:contact.base:readonly`

## 二、硅基流动 API（必须）

### 2.1 注册账号

1. 访问 [硅基流动官网](https://siliconflow.cn/)
2. 注册账号并完成实名认证
3. 进入「API 密钥」页面，创建新的 API Key
4. 记录 API Key → 填写到 `.env` 的 `SILICONFLOW_API_KEY`

### 2.2 充值余额

- 新用户通常有免费额度
- 建议充值 ¥50-100 用于测试和初期使用
- 推荐模型：`Qwen/Qwen2.5-7B-Instruct`（性价比高）

### 2.3 确认模型可用

在硅基流动控制台确认以下模型可用：
- `Qwen/Qwen2.5-7B-Instruct`（默认，推荐）
- 或 `Qwen/Qwen2.5-72B-Instruct`（效果更好，费用更高）

## 三、服务器（必须）

### 3.1 服务器要求

| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 40 GB | 100 GB |
| 操作系统 | Ubuntu 20.04+ / CentOS 7+ | Ubuntu 22.04 LTS |
| 网络 | 公网 IP | 公网 IP + 域名 |

### 3.2 服务器准备

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# 安装 Docker Compose
sudo apt install docker-compose-plugin  # Ubuntu
# 或
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 3.3 域名和 HTTPS（推荐）

飞书事件订阅支持 HTTP，但生产环境强烈建议配置 HTTPS：

1. 购买域名，解析到服务器 IP
2. 申请 SSL 证书（推荐 Let's Encrypt 免费证书）：
   ```bash
   sudo apt install certbot
   sudo certbot certonly --standalone -d your-domain.com
   ```
3. 修改 `config/nginx.conf` 配置 HTTPS

### 3.4 防火墙配置

开放以下端口：
- `80` — HTTP（Nginx）
- `443` — HTTPS（如配置了 SSL）
- `8000` — 后端 API（可选，仅调试用）

## 四、GitHub 仓库（可选，用于开源）

1. 在 GitHub 创建新仓库
2. 确保 `.gitignore` 已排除 `.env` 和 `config/config.yaml`
3. 推送代码前检查无敏感信息

## 五、配置清单汇总

部署前请确认以下所有配置项已填写到 `.env` 文件：

```env
# ✅ 飞书配置（从飞书开放平台获取）
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_VERIFICATION_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_ENCRYPT_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ✅ 硅基流动（从硅基流动控制台获取）
SILICONFLOW_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ✅ 数据库密码（自定义，建议16位以上随机字符串）
DB_PASSWORD=your_strong_random_password_here
```
