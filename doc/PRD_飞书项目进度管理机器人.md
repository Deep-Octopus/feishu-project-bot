# 飞书项目进度管理机器人 PRD

## 1. 项目概述

### 1.1 产品定位
一个基于飞书开放平台的智能项目管理机器人，通过群聊交互方式自动收集、记录和更新项目进度，生成项目报表，提升团队协作效率。

### 1.2 目标用户
- 项目经理/负责人
- 开发团队成员
- 外部协作人员（供应商、外包等）

### 1.3 核心价值
- 降低项目进度跟踪成本，无需额外填写表格
- 自动化生成日报、周报，减少重复劳动
- 实时更新项目进度，提高透明度
- 历史记录可追溯，便于复盘和问题定位

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 日报记录功能
**功能描述**：团队成员通过@机器人提交当日工作内容

**交互方式**：
```
@机器人 【日报】
今天完成了用户登录模块的开发
- 实现了JWT认证
- 完成了单元测试
- 遇到问题：Redis连接超时，已解决
```

**功能要点**：
- 支持@机器人触发
- 识别【日报】关键词
- 自动提取提交人、提交时间
- 支持多行文本、列表格式
- 支持企业外部人员提交

**数据存储**：
- 提交人姓名/ID
- 提交时间（精确到分钟）
- 工作内容（原始文本）
- 关联项目ID
- 消息ID（用于追溯）

#### 2.1.2 项目进度更新功能
**功能描述**：根据日报内容自动更新项目进度表

**更新逻辑**：
1. 解析日报内容，提取关键信息（任务名称、完成状态、问题等）
2. 匹配项目规划中的任务项
3. 更新任务完成度、负责人、最新进展
4. 标记风险项（如"遇到问题"、"延期"等关键词）

**进度表结构**：
| 任务模块 | 任务名称 | 负责人 | 计划完成时间 | 当前状态 | 完成度 | 最新进展 | 风险标记 |
|---------|---------|--------|-------------|---------|--------|---------|---------|

**更新规则**：
- 识别任务关键词（如"完成"、"开发中"、"测试"等）
- 自动计算完成度（基于任务拆分和提交频次）
- 风险自动标记（识别"问题"、"阻塞"、"延期"等）

#### 2.1.3 周报生成功能
**功能描述**：自动汇总一周内的所有日报，生成周报

**交互方式**：
```
@机器人 【周报】
或
@机器人 【周报】@张三  // 生成指定成员的周报
```

**周报内容**：
- 时间范围（本周一至周日）
- 团队整体完成情况
- 各成员工作汇总
- 本周完成的任务列表
- 遇到的问题及解决情况
- 下周计划（基于项目规划）

**输出格式**：
- 飞书消息卡片形式
- 支持导出为文档（上传到群文件）

#### 2.1.4 进度查询功能
**功能描述**：快速查询项目当前进度

**交互方式**：
```
@机器人 【进度】
@机器人 【进度】用户模块
@机器人 【进度】@张三
```

**返回内容**：
- 项目整体完成度
- 各模块完成情况
- 当前风险项
- 最近更新时间

#### 2.1.5 历史记录查询功能
**功能描述**：查询历史日报和项目变更记录

**交互方式**：
```
@机器人 【历史】2024-03-01
@机器人 【历史】@张三 最近7天
@机器人 【历史】用户模块
```

**查询维度**：
- 按日期查询
- 按成员查询
- 按模块查询
- 按时间范围查询

### 2.2 扩展功能

#### 2.2.1 任务提醒功能
**功能描述**：根据项目规划自动提醒任务截止时间

**提醒规则**：
- 任务截止前3天提醒
- 任务截止前1天再次提醒
- 任务逾期每天提醒

**交互方式**：
```
@机器人 【提醒设置】开启/关闭
@机器人 【提醒设置】提前3天
```

#### 2.2.2 项目统计功能
**功能描述**：生成项目数据统计报表

**交互方式**：
```
@机器人 【统计】
@机器人 【统计】本月
```

**统计内容**：
- 团队成员活跃度（日报提交频次）
- 任务完成速率
- 问题数量统计
- 模块进度对比

#### 2.2.3 智能问答功能
**功能描述**：基于项目文档和历史记录回答问题

**交互方式**：
```
@机器人 【问答】用户登录模块谁负责？
@机器人 【问答】上周遇到了哪些问题？
```

#### 2.2.4 会议纪要关联功能
**功能描述**：记录项目会议决议并关联到进度

**交互方式**：
```
@机器人 【会议】
今日会议决议：
1. 用户模块延期到3月25日
2. 增加支付模块优先级
```

**功能要点**：
- 自动更新项目规划
- 记录决议历史
- 关联到进度变更

#### 2.2.5 风险预警功能
**功能描述**：自动识别项目风险并预警

**预警规则**：
- 连续3天无日报提交
- 任务进度长期无更新
- 多次提及"问题"、"阻塞"
- 任务完成度低于预期

**预警方式**：
- 群内@相关人员
- 发送预警消息卡片

## 3. 技术需求

### 3.1 飞书开放平台能力
- 机器人消息接收与发送
- @提及事件监听
- 群文件读取权限
- 企业外部人员权限配置
- 消息卡片渲染
- 定时任务（提醒功能）

### 3.2 数据存储
**项目配置数据**：
- 项目基本信息（名称、描述、开始时间）
- 项目规划（任务列表、负责人、计划时间）
- 群聊ID与项目ID映射

**运行时数据**：
- 日报记录（全量历史）
- 进度表（实时更新）
- 会议纪要
- 提醒配置

**存储方案**：
- 数据库：MySQL/PostgreSQL（结构化数据）
- 文档存储：飞书云文档/本地文件（项目规划、周报）
- 缓存：Redis（高频查询数据）

### 3.3 AI能力（可选）
- NLP文本解析（提取任务、状态、问题）
- 任务匹配算法（日报内容与项目规划匹配）
- 智能问答（基于RAG）

```
✅ 日报已记录
提交人：张三

已自动更新项目进度
[查看进度] [查看历史]
```

**周报卡片**：
```
📊 本周项目周报（3月14日-3月20日）

本周完成任务：5项
遇到问题：2项（已解决1项）

```

```

建议：及时跟进任务进展
```

### 4.3 错误处理
- 命令格式错误：提示正确格式
- 权限不足：提示联系管理员
- 数据异常：记录日志并提示用户
- 网络超时：自动重试3次

## 5. 数据模型

### 5.1 项目表（projects）
```sql
id: 项目ID
name: 项目名称
description: 项目描述
group_id: 飞书群ID
start_date: 开始日期
end_date: 计划结束日期
status: 状态（进行中/已完成/已暂停）
created_at: 创建时间
updated_at: 更新时间
```

### 5.2 任务表（tasks）
```sql
id: 任务ID
project_id: 项目ID
module: 所属模块
name: 任务名称
assignee: 负责人
plan_start: 计划开始时间
plan_end: 计划结束时间
status: 状态（未开始/进行中/已完成/已延期）
progress: 完成度（0-100）
latest_update: 最新进展
risk_flag: 风险标记
created_at: 创建时间
updated_at: 更新时间
```

### 5.3 日报表（daily_reports）
```sql
id: 日报ID
project_id: 项目ID
user_id: 提交人ID
user_name: 提交人姓名
content: 日报内容
submit_time: 提交时间
message_id: 飞书消息ID
is_external: 是否外部人员
created_at: 创建时间
```

### 5.4 进度变更记录表（progress_logs）
```sql
id: 记录ID
task_id: 任务ID
change_type: 变更类型（进度更新/状态变更/风险标记）
old_value: 旧值
new_value: 新值
trigger_by: 触发人
trigger_source: 触发来源（日报/会议/手动）
created_at: 创建时间
```

## 7. 风险与挑战

### 7.1 技术风险
- 飞书API限流：需要做好请求频率控制
- 外部人员权限：需要申请企业外部权限
- 数据一致性：进度更新需要事务保证

### 7.2 产品风险
- 用户习惯培养：需要引导用户使用机器人
- 命令记忆成本：提供帮助文档和快捷入口
- 误识别问题：日报解析可能不准确，需要人工校验机制

### 7.3 解决方案
- 提供详细的使用文档和示例
- 设置管理员审核机制
- 支持手动修正进度
- 定期收集用户反馈优化

## 9. 附录

### 9.1 项目初始化流程
1. 管理员在群内@机器人发送【初始化】
2. 机器人引导上传项目规划文档（Excel/飞书表格）
3. 机器人解析文档，创建项目和任务数据
4. 发送初始化成功消息，展示项目概览

### 9.2 项目规划文档模板
**必需字段**：
- 任务模块
- 任务名称
- 负责人
- 计划开始时间
- 计划结束时间

**可选字段**：
- 任务描述
- 依赖任务
- 优先级

### 9.3 权限配置说明
**机器人所需权限**：
- 接收群消息
- 发送群消息
- 读取群文件
- 获取群成员信息
- 接收@事件
- 企业外部人员交互权限
